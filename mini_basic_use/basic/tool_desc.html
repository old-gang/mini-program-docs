# 开发工具界面组成

  • 菜单栏 切换项目等操作

  • 模拟器 用于展示编译后的效果

  • 编辑器 用于编辑代码(也可使用其他编辑器如 vscode 替代)

  • 调试器 类似于 chrome 的调试器

  • 工具栏

    • 模拟器 控制是否使用模拟器
    • 编辑器 控制是否使用编辑器
    • 调试器 用于调试

      注意: 上面三个部分,至少保留一个

    • 可视化 可以实现拖拽组件等操作,快速搭建页面(一般不使用,代码为王)
    • 云开发 (我们需要为小程序的数据搭建服务器,但是搭建服务器比较繁琐,使用云开发可以省去我们自己搭建服务器的步骤,不过这里不做介绍,因为云开发主要涉及后端开发知识)
    • 小程序模式 两种模式: 小程序模式和插件模式 选择小程序模式
    • 普通编译 可以选择编译模式(进入小程序的方式)
    • 编译按钮 点击则重新编译代码(默认保存代码则自动编译,但有时编辑器会出现卡顿,需要手动编译)
    • 预览按钮 可以在真机上查看页面效果
    • 真机调试按钮 在真机上预览页面, 弹出调试窗口
    • 清缓存按钮
    • 上传按钮 可以将项目上传给腾讯进行审核,审核通过则上线
    • 版本管理按钮 利用 git 实现管本管理
    • 详情按钮 展示或修改项目信息
    • 消息按钮 微信小程序推送的通知

# 编辑器项目目录介绍

├── pages 存放小程序页面的目录
│   ├── index  index页面目录
│   │   ├── index.js  index页面的逻辑文件
│   │   ├── index.json index页面的配置文件
│   │   ├── index.wxml index页面的结构文件
│   │   └── index.wxss index页面的样式文件
│   ├── logs
│   │   ├── logs.js
│   │   ├── logs.json
│   │   ├── logs.wxml
│   │   └── logs.wxss
├── utils
│   └── utils.js 存放工具函数
├── .eslintrc.js eslint配置文件
├── app.js 全局逻辑文件 App() 必须在 app.js 中调用，必须调用且只能调用一次
├── app.json 全局配置文件
├── app.wxss 全局样式文件
├── project.config.json 项目的配置文件
├── project.private.config.json 项目私有配置文件。此文件中的内容将覆盖 project.config.json 中的相同字段。项目的改动优先同步到此文件中
└── sitemap.json 用来配置小程序及其页面是否允许被微信索引(小程序是否可以在微信中被检索)
