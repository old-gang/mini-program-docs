# 小程序的状态管理

把小程序中多个组件或页面用到的共享数据集中管理起来的方案

小程序中实现状态管理,是利用mobx-miniprogram和mobx-miniprogram-bindings这两个第三方包实现的

  • mobx-miniprogram 用于创建管理共享数据的仓库
  • mobx-miniprogram-bindings 用于将共享数据仓库和小程序的组件或页面连接起来

# 使用

# 创建 store 对象

  • 下包: npm install mobx-miniprogram mobx-miniprogram-bindings. 重新构建 npm

  • 创建 store 对象

    // /model/store.js
import { observable, action } from 'mobx-miniprogram'
export const store = observable({
  // 数据字段
  count: 1,
  // 计算属性
  get sum() {
    return this.count + 8
  },
  // actions(方法)
  add: action(function () {
    this.count = this.count + 1
  }),
  sub: action(function () {
    this.count = this.count - 1
  }),
})

此时集中管理数据的仓库已经创建完毕,接下来,我们在页面和组件中操作仓库中的数据

# 在页面中使用

  1. 创建 behavior

    // /model/behavior.js

// BehaviorWithStore 用于创建和store绑定的behavior
import { BehaviorWithStore } from 'mobx-miniprogram-bindings'
// 引入store
import { store } from './store'
// 创建和store绑定好的behavior
export const countBehavior = BehaviorWithStore({
  // behavior和store绑定的配置项
  storeBindings: [
    {
      store: store, //绑定的store
      fields: ['count', 'sum'], // 绑定store中的数据字段,在引入当前behavior的页面中可以直接使用
      // 绑定store中的add函数,在引入当前behavior的页面中可以直接使用
      actions: ['add'],
    },
  ],
})

  2. 在页面中引入使用 countBehavior

    // /pages/index/index.js
// 引入countBehavior
import { countBehavior } from '../../model/behavior'
Page({
// 配置countBehavior
behaviors: [countBehavior],
data: {},
})

     
     

     
     


  3. 在页面的 wxml 中直接使用数据和方法

    <view>index页面</view>
<view>
  <view>count: {{count}}</view>
  <view>sum: {{sum}}</view>
  <button size="mini" bindtap="add">+</button>
</view>

  4. 效果

# 在组件中使用

  1. 在组件的 js 文件中配置和 store 绑定的代码

    // /components/my-comp/my-comp.js
// 引入一个和store绑定并注册组件的函数
import { ComponentWithStore } from 'mobx-miniprogram-bindings'
// 引入store
import { store } from '../../model/store'
// 注册一个和store绑定的组件
ComponentWithStore({
  // data 组件自己的数据
  data: {
    msg: 'my-comp自己的数据',
  },
  // 组件和store进行绑定
  storeBindings: {
    store: store,
    fields: ['count', 'sum'],
    // actions: ["sub"] 这样写可以或写成下面的写法
    actions: {
      // subTap表示my-comp中的函数,subTap触发则调用store中的sub函数
      subTap: 'sub',
    },
  },
})

  2. 在组件结构中使用数据和方法

    <!-- /components/my-comp/my-comp.wxml -->
<view style="border: 1rpx solid #ccc;">
  <view>my-comp组件</view>
  <view>
    <view>msg: {{msg}}</view>
    <view>count: {{count}}</view>
    <view>sum: {{sum}}</view>
    <button size="mini" bindtap="subTap">-</button>
  </view>
</view>

  3. 效果

以上是 mobx 和页面以及组件绑定的新语法,旧语法可以参考文档: mobx-miniprogram-bindings 旧版语法open in new window
